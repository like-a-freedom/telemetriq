# Plan: Hide Unavailable Telemetry Data on Preview Page

## Problem
On the preview page, some template overlays show placeholder values for unavailable telemetry data:
- **Trail Run layout**: Shows `'--'` for missing HR, Grade, Elevation
- **Cycling Pro layout**: Shows `'N/A'` + `'NO SENSOR'` for missing Cadence, Power, HR

The user wants these metrics to simply not appear when their data is unavailable.

## Changes

### 1. `frontend/src/modules/layouts/trailRunLayout.ts`
- Filter the `columns` array to exclude metrics where the raw frame value is `undefined` (currently hardcodes `'--'`)
- Only include a column if the corresponding frame field (`frame.hr`, `frame.gradePercent`, `frame.elevationM`) is defined
- The existing `columnWidth = w / columns.length` will automatically redistribute space

### 2. `frontend/src/modules/layouts/cyclingProLayout.ts`
- Filter `SIDEBAR_METRICS` to only iterate over metrics where `frame[key] !== undefined`
- Adjust `backdropBottom` to be calculated from actual rendered metric count instead of always 3
- The existing layout math (`top + index * metricBlockHeight`) will naturally adjust since only available metrics are rendered
